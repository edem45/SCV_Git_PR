# Работа с GIT и GITHUB

## 1. Проверка наличия установленного GIT
В терминале выполнить команду `git version`

Если Git установлен появится сообщениие с информацией о версии программы иначе будет сообщение об ошибке.

## 2. Установка Git
Загружаем последнюю версию Git с сайта https://git-scm.com/download.

Устанавливаем с настройками по умолчанию.

## 3. Настройка Git
При первом использовании Git на необходимо представиться. Для этого нелбходимо ввести в терминале две команды:

 ```git config --global user.name «Ваше имя английскими буквами» ```

```git config --global user.email ваша почта@example.com ```

## 4. Иициализация репозитория 
Получить репозиторий можно двумя способами:
1. В терминале перейти к папке в которой хотим создать репозиторий. Выполняем команду:
```
git init
```
В исходной папке появится скрытая папка **.git**

2. Клонировать существующий репозиторий Git с любого места. Сделать можно это с помощью команды:
```
git clone <адрес или ссылка>
```
## 5. Добавление файлов в область подготовленных и апись изменений в репозиторий 

Для того чтобы сохранить изменения в репозитории неоюходимо выполнить следующие пункты:

|1.Команда *CTRL+S*|2.Команда *git add <имя файла>*|3.Команда *git commit -m <Примечание>*|4.Команда *git commit -a -m <Примечание>*|
|:---:|:---:|:---:|:---:|
|После каждого изменения файла необходимо сохранить его использовав комбинацию *CTRL+S* либо настроить *АВТОСОХРАНЕНИЕ*|Добавить отдельный файл в область подготовленных файлов можно параметром add с указанием имени файла|При создании коммита в репозитории можно добавить однострочное сообщение с помощью параметра commit с флагом -m. Само сообщение вводится непосредственно после флага, в кавычках.|Также можно использовать комбинированную команду *git commit -a -m <Примечание>* которая в свою очередь добавляет файл и фиксирует изменения|


## 6. Проверка статуса репозитория
Просмотреть статус нужного репозитория можно по ключевому слову status: его действие распространяется на подготовленные, неподготовленные и неотслеживаемые файлы.
```
git status
```
## 7. Просмотр истории коммитов с изменениями

Просматривать изменения можно с помощью команды:
```
git log
```
Он отображает список последних коммитов в порядке выполнения.
Кроме того:
> Добавив **--oneline** вы можете посмотреть сокращенный вывод содержащий одну строку с сокращенным хешом.

> Добавив флаг **-p** вы можете подробно изучить изменения, внесённые в каждый файл.

## 8. Просмотр заданного коммита 

Просмотреть полный список изменений, внесённых конкретным коммитом, можно с помощью параметра show, указав идентификатор или хеш коммита. Значение хеша уникально для каждого коммита, созданного в вашем репозитории.
```
git show <хеш>
```
либо
```
git checkout <хеш>
```

## 9. Возврат к актуальному состоянию и продолжение работы

Необходимо ввести след команду:
***
```
git checkout master
```

## 10. Просмотр изменений до коммита 

Необходимо ввести след команду:
```
git diff
```
Для просмотра подготовленных изменений необходимо добавить флаг --staged
```
git diff --stages
```

Также можно указать имя файла как параметр и просмотреть изменения, внесённые только в этот файл.
```
git diff <file name>
```

## 12. Перемещение между сохранениями

Для того чтобы перейти от одного сохранения к другому необходимо выполнить следующюю команду:
```
git checkout <первые 5 цифр хеша коммита>
```
## 13. Игнорирование файлов

Для того чтобы исключить из отслеживания в репозитории определенных файлов и папки необходимо создать файл ***.gitignore*** и записать в него их названия или шаблоны соответствующие таким файлам или папкам.

## 14. Создание веток

По умолчанию имя основной ветки в Git - *Master*.

Создать ветку можно командой :
```
git branch <branch_name>
```

Список веток в репозитории можно посмотреть с помощью команды  `git branch` .
Текущая ветка будет отмечена звездочкой:
*\*master*
